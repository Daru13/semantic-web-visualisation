Readable URIs

We built a library to allow developers working with the Semantic Web to better visualise the structure and the statistical properties of a set of URIs (e.g. tables returned by SPARQL queries). Our library is in two folds:
  • we enhanced the table returned by a SPARQL query,
  • we built two visualisations to study the structure of URIs in depth.

Enhancing the table

Our first improvement of the table is how we display the URIs. We chose to display only the important parts of an URI:
  • the domain, i.e. the name of the website, represented in bold to make it pop out,
  • the language of the website, if possible to detect based on the URI alone, represented by a flag
  • the path, i.e. the path to this specific webpage on the website.
In another hand, we also added dashboards at the top of the column which displays basics statistics per column.

Visusalizing the structure of URIs

In order to better understand the structure of URI we built two visualisations.

The first one is a word cloud. It is displaying the keywords of a column organized from the most present to the least. A keywords is either the last part of the path of an URI, i.e. the name of the webpage, or keywords extracted from pure text by regular expressions.

The second one is a Sankey diagram. It has three main columns: the subdomain, the domain plus tld and the path columns. For each elements of those columns (for example wikipedia.org), the more URIs with that element, the thicker and darker the visual representation is. For two elements from two of those columns (for example dbpedia.org and ressource), the more URIs with both elements, the thicker and darker the link between those two elements is.
