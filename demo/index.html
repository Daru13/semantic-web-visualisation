Visualising SPARQL responses

« Readable URIs »
We built a library to help developers working with semantic web data to better visualise the structure, the theme and some statistical properties of SPARQL query responses (tables containing many URIs). We based our work on SPARQL responses from dbpedia.org, and our contribution is twofold:
  1. we enhance the representation of the data table returned by a SPARQL query;
  2. we propose two specialised visualisations to illustrate URI structures and themes.
Links to the demos are listed at the bottom of this page!

Enhancing the data table

We started by improving the representation of URIs in the data table, assuming developers may want to have a look at the actual data returned by a SPARQL query they ran. We chose to hide the repetitive parts (such as the protocol) and to highlight the most meaningful ones for our target audience:

  • the domain of the URI (where is the data located);
  • the country/language of the URI as a flag, when we could extract it from the URI;
  • the path of the URI (what is the name of this piece of data).

In order to handle very long tables, we decided to paginate the table. You can use the page browser at the bottom-right hand corner of your screen or the left/right arrow keys to navigate between pages.

We also added dashboards at the top of each column. They display basics statistics about the content of the whole column and contain buttons to display more specific visualisations (described below).

Visusalising themes and URI structures

When studying a column, a developer may want to focus in one of the two following directions: the main theme of the cells and the plurality of the URI structures. We created two dedicated visulisations to help them perform such tasks.

  • The first one is a word cloud. It is displaying the keywords of a column organized from the most present to the least. A keywords is either the last part of the path of an URI, i.e. the name of the webpage, or keywords extracted from textual cells using regular expressions.

    This visualisations allows to understand the dominant theme in a column. For example, in demo 1, for the genre column, we can read the most present genres of music in the database.

  • The second one is a Sankey diagram. It has three main columns: the subdomain, the domain with the domain extension (TLD), and the path. For each elements of those columns (such as wikipedia.org), the more URIs with that element, the thicker and darker the visual representation is. For two elements from two of those columns (for example dbpedia.org and resource), the more URIs with both elements, the thicker and darker the link between those two elements is.

Demo pages

Given the beta status and the requirements for using dbpedia's REST API, and given the limited time we had, we did not build a form where you could run any SPARQL query you want to visualise.

Instead, we uploaded a few demos based on SPARQL reponses obtained via dbpedia's SPARQL query runner. We used responses with different numbers of rows and/or columns in order to illustrate the robustness of our library.

Link Description of the data
Demo 1 Small subset of the sample SPARQL response about films we were given at the beginning of the project (3 columns, 500 rows).
Demo 2 Large subset of the sample SPARQL response about music we were given at the beginning of the project (3 columns, 50,000 rows).
